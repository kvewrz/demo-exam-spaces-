# Loader: простой и понятный индикатор загрузки

## Идея «одного источника правды»

Для каждого асинхронного действия должно быть ясно: **идёт запрос или нет**. Источником может быть:

- флаг `isLoading` / `isFetching` из **TanStack Query** или **RTK Query**;
- локальный `useState` для мутаций без библиотеки;
- общий контекст «глобальный спиннер» для критичных блокировок UI.

Избегайте расхождения: кнопка «Отправить» не должна оставаться активной, если запрос уже ушёл.

## Варианты отображения

| Вариант | Когда уместно |
|---------|----------------|
| **Overlay** на весь экран | Первый вход в приложение, смена маршрута с тяжёлым запросом. |
| **Inline** спиннер в секции таблицы | Списки с пагинацией — блок списка затемнён или показывается спиннер на месте данных. |
| **Skeleton** | Длинная загрузка контента; пользователь видит каркас макета. |
| **Disabled кнопка** | Формы — всегда при `isSubmitting`. |

## Минимальный пример: CSS-спиннер + условный рендер

```tsx
function Spinner({ label = 'Загрузка…' }: { label?: string }) {
  return (
    <div className="loader" role="status" aria-live="polite">
      <span className="loader__ring" aria-hidden />
      <span className="loader__text">{label}</span>
    </div>
  );
}

/* Пример стилей — компактный круглый индикатор */
/*
.loader { display: inline-flex; align-items: center; gap: 0.5rem; }
.loader__ring {
  width: 1.25rem; height: 1.25rem;
  border: 2px solid #e5e7eb; border-top-color: #2563eb;
  border-radius: 50%; animation: spin 0.7s linear infinite;
}
@keyframes spin { to { transform: rotate(360deg); } }
*/

// Использование:
// {isLoading ? <Spinner /> : <Table rows={data} />}
```

Для доступности у полноэкранного оверлея добавьте `aria-busy="true"` на контейнер страницы на время загрузки.

## Связка с данными

- При **пагинации** показывайте loader **в области таблицы**, чтобы фильтры не прыгали.
- При **отправке формы** — блокировка кнопки и локальный спиннер рядом с текстом «Сохранить».

Итог: один понятный паттерн на проект — пользователь всегда видит, что система реагирует на действие.
