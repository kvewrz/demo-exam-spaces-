# Фильтрация и сортировка: на клиенте и на сервере

## Когда что использовать

| Подход | Когда уместно |
|--------|----------------|
| **Сервер** (для `GET /spaces`: `zoneType`, `minPrice`, `maxPrice`, `minCapacity`, `sort`, `page`, `limit`) | Большие списки, пагинация, источник истины в БД — **основной каталог** в учебном API. |
| **Клиент** (`useMemo`, фильтрация массива уже загруженных данных) | Маленький список целиком в памяти (например «популярное» из отдельного эндпоинта), быстрый прототип, второстепенный виджет. |

Серверная фильтрация **обязательна**, если данные не помещаются на одну страницу или меняются у других пользователей.

## Сервер: один источник истины в URL

Рекомендуется синхронизировать фильтры с **`useSearchParams`** (React Router):

- Пользователь может **поделиться ссылкой** с теми же фильтрами.
- Кнопка «Назад» в браузере возвращает предыдущее состояние фильтров.

Схема:

1. Читаете `searchParams` → начальные значения формы.
2. При изменении фильтра обновляете URL и **сбрасываете `page` на 1** (новый набор строк).
3. Запрос к API использует и фильтры, и `page`, `limit`.

## Клиент: `useMemo`

Если массив `items` уже есть на клиенте:

```tsx
const filtered = useMemo(() => {
  return items.filter((x) =>
    filterZone ? x.zoneType === filterZone : true
  );
}, [items, filterZone]);
```

Не применяйте одновременно **тяжёлую серверную фильтрацию** и ещё одну такую же на клиенте к тем же данным — получите пустые результаты или путаницу. Либо сервер, либо клиент для одного списка — или чётко разделите роли (сервер — страница каталога, клиент — локальный поиск по уже загруженной странице).

## Сортировка без потери фильтров

При смене `sort` отправляйте **все активные фильтры** тем же запросом. Не храните сортировку отдельно от параметров, которые уже ушли на сервер, если список серверный.

## Итог

- Каталог с пагинацией в задании — **фильтры и сортировка на бэке** + отражение в UI и опционально в URL.
- Отдельный небольшой список можно отфильтровать **на фронте** для отработки `useMemo` и понимания разницы подходов.
